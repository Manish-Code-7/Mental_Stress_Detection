# EDA (Exploratory Data Analysis) Dashboard Guide

## Overview

The EDA Dashboard provides comprehensive visualization and analysis of your mental stress detection dataset. It includes:

- **Label Distribution Analysis**: Pie charts, bar charts, and class balance visualization
- **Text Analysis**: Word frequency, text length distributions, and text statistics by label
- **Correlation Analysis**: Feature correlations and high correlation warnings
- **Mental Health Insights**: Automated insights about class balance, text quality, and dataset health
- **Subreddit Analysis**: Community-specific patterns and stress rates

## Accessing the EDA Dashboard

1. **Start the Backend**:
   ```bash
   cd web_files/backend
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   python app.py
   ```

2. **Start the Frontend**:
   ```bash
   cd web_files/frontend
   npm install
   npm run dev
   ```

3. **Navigate to EDA**:
   - Open http://localhost:3000
   - Click "View EDA" or navigate to "/eda"

## Features

### 1. Dataset Metrics Panel
- **Dataset Shape**: Number of rows and columns
- **Memory Usage**: Total memory consumption
- **Duplicates**: Number of duplicate rows
- **Missing Values**: Total missing data points

### 2. Mental Health Insights
Automated insights include:
- **Class Balance Warnings**: Alerts about dataset imbalance
- **Text Quality Assessment**: Analysis of text length and quality
- **Data Quality Recommendations**: Suggestions for preprocessing

### 3. Data Visualizations

#### Distributions Tab
- Label distribution (pie chart)
- Label distribution (bar chart)
- Class balance analysis (horizontal bar)

#### Text Analysis Tab
- Top 20 word frequencies
- Text length distribution (area chart)
- Text statistics by label (box plots)

#### Correlations Tab
- High correlations (|r| > 0.7) between features
- Correlation warnings and recommendations

#### Subreddit Tab
- Subreddit-specific analysis (if available)
- Community stress patterns

## Backend EDA Endpoint

The backend automatically serves EDA data from:
1. **Primary**: `ml_model/reports/eda_results.json` (if generated by notebook)
2. **Fallback**: Auto-generates basic EDA from `ml_model/stress.csv`

### API Endpoint
```
GET /eda
```

## Generating EDA from Notebook

To generate comprehensive EDA using the notebook:

1. **Run the EDA Cell** in `ml_model/model_upgrade.ipynb`:
   - Cell 2: Enhanced EDA & Advanced Visualizations
   - This generates `ml_model/reports/eda_results.json`
   - Also creates visualization PNGs in `ml_model/visualizations/`

2. **The backend will automatically use** the generated EDA data

## Customization

### Adding New Visualizations

1. **Backend**: Add new fields to the EDA response in `app.py`
2. **Frontend**: Create new chart components in `EnhancedEDACharts.tsx`
3. **Update Tabs**: Add new tab content for your visualization

### Styling

- Colors are defined in `EnhancedEDACharts.tsx`:
  ```typescript
  const STRESS_COLORS = {
    high_stress: "#FF6B6B",
    low_stress: "#4ECDC4",
    neutral: "#95E1D3",
    positive: "#38B6A8",
  };
  ```

## Troubleshooting

### No EDA Data Available
- **Check**: Ensure `ml_model/stress.csv` exists
- **Check**: Backend can access the file (correct paths)
- **Solution**: Backend will auto-generate basic EDA from CSV

### Missing Visualizations
- **Check**: Browser console for errors
- **Check**: Backend `/eda` endpoint returns data
- **Solution**: Verify data structure matches frontend expectations

### Backend Not Connected
- **Check**: Backend is running on port 8001
- **Check**: Frontend `BACKEND_URL` environment variable
- **Solution**: Verify CORS is enabled in backend
